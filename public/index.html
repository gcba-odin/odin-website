¿QUIÉNES SOMOS?

Una plataforma de código abierto y de uso libre, desarrollada por el Laboratorio de Innovación del Gobierno de la Ciudad de Buenos Aires para publicar y habilitar el uso y descarga de catálogos de datos.

@odin_org

¿POR QUÉ ODÍN?

El nombre es un homenaje al dios nórdico de la sabiduría. Odín desde su trono podía ver todo lo que pasaba en los nueve mundos y tenía dos cuervos que le transmitían el conocimiento que obtenían volando.

Esta plataforma comparte con el dios mitológico el valor del conocimiento, la disposición al entendimiento y la constante búsqueda de información y herramientas que ayudan a su organización.

Abre catálogos de datos generados por el sector privado, el público y el tercer sector fomentando transparencia gubernamental y participación de los ciudadanos. Además impulsa el desarrollo de políticas, productos y servicios basados en datos e innovación.

USABILIDAD

- La plataforma cuenta con un administrador que hace más fácil e intuitivo el manejo del contenido.

- La estética de la aplicación es amable al ciudadano, le permite encontrar datos de manera fácil y rápida.


NUEVO CONCEPTO DE DATASET

- Un dataset que engloba todos los recursos.

- Los archivos son almacenados en la base de datos, permitiendo buscar datasets por el contenido de los archivos.

- Los datasets y recursos tienen una propiedad llamada estado que hace más fácil publicar y ocultar información.

- Integra campos por defecto, lo cual permite su búsqueda, filtrado y tratamiento.

- Mayor granularidad en la actualización de la información reflejandose en las fechas de los recursos según la entidad actualizada.

CALIDAD DE DATOS

- Permite normalizar organizaciones.

- Mejor presentación de la información.

- Genera mapas tanto desde un CSV, XLS, XLSX, JSON o embebidos.

- Permite incluir gráficos desde plataformas que generen iframes.

- Un mismo archivo puede mostrarse de diferentes formatos: mapa, gráfico, tabla y/o pdf.

MEJORAS TÉCNICAS

- La arquitectura utilizada permite crear diferentes aplicaciones utilizando la API.

- El contenido de los archivos se guarda en una base de datos para usarlo y consultarlo.

¿CÓMO FUNCIONA?

Mediante un administrador que controla y publica datos y una API que permite el acceso a información desde aplicaciones externas. Odín se encuentra bajo la licencia GPL.

Tecnologías para el desarrollo de Odín:

- API Rest creada con Sails.js, framework de NodeJS.

- Frontend y administrador creados sobre AngularJS 1.5.x, HTML5 y CSS3. Gulp y Bower para la automatización, minificación y descarga de dependencias.

- Base de datos NoSQL (MongoDB) para la persistencia de los archivos.

- Base de datos relacional (PostgreSQL) para guardar la metadata de todas las entidades. Odín convierte automáticamente los archivos subidos de tipo CSV, XLS, XLSX a JSON para poder ver su contenido, aplicar filtros y búsquedas, generar paginación, previsualizaciones, etc.

La API Rest utiliza Kong (getkong.org) como Gateway para manejar autenticación, limite de tráfico, ACL, CORS y relacionados, por lo que se necesitará una ApiKey para las interacciones con la API.
